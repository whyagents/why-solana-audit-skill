# WHY Solana Audit Skill

When this repository is installed, use the `why-solana-audit` skill for evidence-driven Solana security audits.

## Operating Rules

- Inspect the repo before judging it.
- Cite files and functions for every audit claim.
- Prefer high-confidence findings over broad speculation.
- Separate Solana program risks from client, backend, indexer, infrastructure, and operational risks.
- For High or Critical DeFi claims, require a runnable proof, invariant test, local fork trace, or transaction-level simulation.
- Label mock-only behavior as mock-dependent until reproduced against realistic behavior or deployed code.
- Keep reports actionable for founders and engineers.

## Suggested Commands

- `/why-audit`
- `/why-audit-report`
