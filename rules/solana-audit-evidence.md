# Solana Audit Evidence Rule

Apply these constraints whenever reviewing Solana code.

- Read before judging.
- Cite files and functions for every claim.
- Do not assign High or Critical severity to DeFi claims without proof-quality evidence.
- Label mock-only issues as mock-dependent.
- Run a DeFi exploit-class pass for financial protocols and integrations.
- Give reasons for not-applicable exploit classes.
- Separate confirmed findings from plausible risks and open questions.
- Prefer fewer high-confidence findings over noisy speculation.
